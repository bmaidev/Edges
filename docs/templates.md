# Templates Catalog

Templates are one-tap, research-grounded session sequences. Each is a ready-made flow of phases (a "session") that you launch instead of building one from scratch.

**Where to use them**

- **Host console session picker** — pick a template and launch it in one tap.
- **Builder** (`/r/<room>/build`) — load a template as a starting point, then tweak it before you run it.

**A note on AI templates**

Some templates use AI features (a devil's advocate, persona panels, live prototyping, and so on). These are flagged below as **needs an AI key**. They require the `ANTHROPIC_API_KEY` to be configured on the server. Where a template can still run without one, that's noted too (most degrade gracefully).

There are **21 templates**, grouped below by what they're for: **Decide**, **Diverge**, **Reflect**, **Dialogue**, and **AI**.

## At a glance

### Decide
For prioritising, committing, and making choices.

| Template | What it does |
|---|---|
| Pre-Mortem | Prospective hindsight: imagine the project already failed, surface the causes, prioritise the real risks. ~30% more risks than asking 'what could go wrong'. |
| 15% Solutions | Strip ambition to what each person can do THIS WEEK with no new budget or authority — turns vague enthusiasm into owned action. |
| Idea Marketplace | Generate ideas, then invest a budget across them to predict which will succeed — aggregates dispersed judgement better than a show of hands. |
| 25/10 Crowd Sourcing | Everyone writes one bold idea; ideas detach from authors and get blind-scored over five passes; the top ten surface. Beats a show of hands. |
| Min Specs | List every rule we think we must follow, then ruthlessly subtract — keep only the few whose removal would actually cause failure. |

### Diverge
For generating ideas and opening up thinking.

| Template | What it does |
|---|---|
| Anti-Problem (reverse brainstorm) | Ask how to GUARANTEE failure — taboo truths surface in a playful frame — then invert into what to actually do. |
| Six Thinking Hats (speed run) | Parallel thinking — everyone wears the same hat at once, so critique stops feeling like attack. Six fast timed passes. |
| 1-2-4-All | The same question worked alone → in pairs → in fours → whole group. The workhorse Liberating Structure for inclusive idea generation. |
| Silent Brainwrite | Silent round-robin building — no talking, no production-blocking, no dominance. Reliably out-produces verbal brainstorming. |

### Reflect
For debriefs, retros, and pulse checks.

| Template | What it does |
|---|---|
| What³ (What / So What / Now What) | A clean debrief arc: separate observation from interpretation from action, in three timed passes. |
| Rose / Thorn / Bud | A fast retro: what's working (rose), what hurts (thorn), what's an emerging opportunity (bud). |
| Psychological-Safety Pulse | An anonymous start-and-end read on team safety (Edmondson's items), so you can see whether the session itself moved the needle. |

### Dialogue
For structured conversation across the whole room.

| Template | What it does |
|---|---|
| World Café | Small tables explore one question; hosts stay, everyone else scatters to a new table each round — ideas cross-pollinate across the room. |
| Shift & Share | Intact small groups tour a set of stations, one round each — a fast way to share many parallel ideas without death-by-presentation. |

### AI
Use AI in the room. **All need an AI key** (`ANTHROPIC_API_KEY`).

| Template | What it does |
|---|---|
| AI Red-Team a Plan | State the plan, let the AI devil's-advocate attack the room's consensus, then take a consent temperature. |
| Blind-Customer Test | Pitch the idea, then a panel of AI personas reacts in character so you can pressure-test it before building. |
| The Empty Chair | Give the absent stakeholder a voice — the room interviews an AI standing in for a customer, regulator, or future user. |
| Live Issue Map | The room contributes freely; the AI organises it into issues and the positions people hold, and the facilitator focuses the room one issue at a time. |
| Prompt Relay | The room co-builds one AI prompt — each person adds a constraint, audience, tone, or example — then runs it together. Builds shared prompt literacy. |
| Prototype It Live | Describe an interface in words; the AI builds a clickable prototype the room can react to in seconds. |

## Decide

### Pre-Mortem
**For:** surfacing and prioritising the risks to a project before you commit. Use it at the start of an initiative, when the room is optimistic and you want the unspoken doubts on the table.

Lobby → The failure headline → Read the risks → How likely / how bad? → Close

### 15% Solutions
**For:** turning a good discussion into concrete, owned next steps. Use it at the end of a session, or whenever enthusiasm is high but nobody has said what they'll actually do.

Lobby → Your 15% → Commitments read-around → Close

### Idea Marketplace
**For:** choosing between options by tapping the room's collective judgement. Use it when you have several candidate ideas and want a sharper signal than a show of hands. Everyone gets 100 credits to invest (up to 50 per idea).

Lobby → Put ideas on the table → Invest → Close

### 25/10 Crowd Sourcing
**For:** surfacing the boldest ideas and ranking them fairly. Use it in a larger group where you want anonymous, author-blind scoring. Ideas are scored over five passes.

Lobby → 25/10 → Close

### Min Specs
**For:** stripping a plan down to its essential rules. Use it when a process feels over-constrained and you want to find the handful of "musts" that actually matter.

Lobby → Min Specs → Close

## Diverge

### Anti-Problem (reverse brainstorm)
**For:** breaking a stuck or polite group open by flipping the question. Use it when direct brainstorming has stalled — asking how to guarantee failure surfaces honest problems, then you invert them into actions.

Lobby → How do we guarantee failure? → Read the sabotage → Now invert → Close

### Six Thinking Hats (speed run)
**For:** examining one topic from every angle without the group fragmenting into argument. Use it for a decision or idea that deserves a thorough, multi-perspective look. Six fast timed passes.

Lobby → ⚪ White — facts → 🔴 Red — feelings → ⚫ Black — risks → 🟡 Yellow — benefits → 🟢 Green — ideas → 🔵 Blue — next → Close

### 1-2-4-All
**For:** inclusive idea generation where everyone contributes, not just the loudest voices. The reliable default for opening up almost any question. Works the same prompt alone, in pairs, in fours, then as a whole group.

Lobby → 1-2-4-All → Close

### Silent Brainwrite
**For:** generating a high volume of ideas quickly and equitably. Use it when a few people tend to dominate, or when ideas need room to breathe. Everyone builds in silence, passing cards round-robin.

Lobby → Build in silence → Close

## Reflect

### What³ (What / So What / Now What)
**For:** a clean, structured debrief after an event or piece of work. Use it to keep observation, meaning, and action from blurring together.

Lobby → What? → So what? → Now what? → Next steps read-around → Close

### Rose / Thorn / Bud
**For:** a fast, lightweight retrospective. Use it as a regular team check-in: what's working, what hurts, what's emerging.

Lobby → 🌹 Rose → 🌵 Thorn → 🌱 Bud → Read-around → Close

### Psychological-Safety Pulse
**For:** measuring team safety and whether the session moved it. Use it as an anonymous before-and-after wrapper around important work. The same three Edmondson items are asked at the start and the end.

Lobby → Pulse — start → What would make it safer? → Pulse — end → Close

## Dialogue

### World Café
**For:** cross-pollinating ideas across a larger room. Use it for an open, exploratory question where you want every table's thinking to mix. Hosts stay at each table; everyone else moves between rounds.

Lobby → Café → Close

### Shift & Share
**For:** sharing many parallel ideas without long presentations. Use it when there are several things to show or react to — groups tour the stations, one round each.

Lobby → Stations → Close

## AI

> Every template in this section needs an AI key (`ANTHROPIC_API_KEY`) configured on the server.

### AI Red-Team a Plan
**For:** pressure-testing a plan before committing to it. Use it when the room is converging and you want a hard, impartial challenge before locking in. **Needs an AI key; degrades to a plain consent check without one.**

Lobby → The plan → Devil's advocate → Where do we land? → Close

### Blind-Customer Test
**For:** getting a fast reaction to an idea from the people it's meant to serve. Use it to stress-test a pitch before you build. **Needs an AI key.**

Lobby → The pitch → Customer panel → Close

### The Empty Chair
**For:** bringing an absent stakeholder's perspective into the room. Use it when a customer, regulator, or future user can't be present but their voice matters. **Needs an AI key.**

Lobby → Empty chair → Close

### Live Issue Map
**For:** making sense of a messy, multi-sided discussion in real time. Use it when many issues are tangled together and you want to focus the room one at a time. **Needs an AI key.**

Lobby → What's on your mind? → Issue map → Close

### Prompt Relay
**For:** building shared AI prompt literacy as a group. Use it to co-write one prompt — each person adds a constraint, audience, tone, or example — then run it together. **Needs an AI key.**

Lobby → Build the prompt → Close

### Prototype It Live
**For:** turning a described interface into something clickable in seconds. Use it to react to a real screen instead of arguing over words. **Needs an AI key.**

Lobby → Describe the screen → Build → Close

## Building your own

You don't have to start from a template. In the builder (`/r/<room>/build`) you can assemble a custom phase sequence from scratch, and the builder's **"Suggest a session"** can generate one for you from a goal you describe.
